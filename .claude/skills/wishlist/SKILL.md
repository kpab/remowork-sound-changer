# Wishlist Management

## 概要

ウィッシュリスト（機能要望・改善案）の管理スキル。
今後実装したい機能や改善案を記録・追跡する。

## 使用タイミング

- 新しい機能要望を記録する時
- 今後の改善案を追加する時
- ウィッシュリストを確認する時
- 優先度を変更する時

## ウィッシュリストの場所

各プロジェクトの `WISHLIST.md` に記載：
```
remowork-sound-changer/
└── WISHLIST.md
```

## フォーマット

```markdown
# Wishlist

機能要望・改善案リスト

## 優先度: 高
- [ ] 機能名 - 説明 (追加日: YYYY-MM-DD)

## 優先度: 中
- [ ] 機能名 - 説明 (追加日: YYYY-MM-DD)

## 優先度: 低
- [ ] 機能名 - 説明 (追加日: YYYY-MM-DD)

## 完了済み
- [x] 機能名 - 説明 (完了日: YYYY-MM-DD, バージョン: v1.x.x)
```

## 操作

### 追加
1. WISHLIST.md を開く
2. 適切な優先度セクションに追記
3. 追加日を記載

### 完了
1. 該当項目を「完了済み」セクションに移動
2. チェックを付ける `[x]`
3. 完了日とバージョンを記載

### 確認
1. WISHLIST.md を読む
2. 優先度順にリスト表示
